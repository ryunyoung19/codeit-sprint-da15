### Q1. 훈련 정확도 95%, 테스트 정확도 65% — 이 현상은?

- **과적합 (Overfitting)** : 훈련 데이터의 노이즈와 세부사항까지 암기해버려 새로운 데이터에 대응하지 못하는 현상
- **주요 원인**
    - 모델 복잡도가 지나치게 높을 때
    - 훈련 데이터 양이 너무 적을 때
    - 학습을 너무 오래 반복했을 때 (epoch 과다)
- **대처 방법**

| 방법 | 설명 |
| --- | --- |
| 데이터 증강 | 훈련 데이터를 더 많이 확보하거나 다양성을 높인다 |
| 규제 (L1/L2) | 특정 가중치에 과도하게 의존하지 못하도록 제한한다 |
| 모델 단순화 | 불필요하게 복잡한 모델 대신 단순한 모델을 선택한다 |
| 교차 검증 | K-Fold로 다양한 분할에서 성능을 검증해 조기에 감지한다 |
| 조기 종료 | 검증 성능이 개선되지 않으면 학습을 중단한다 |

### Q2. 회귀 모델 평가 시 MSE vs MAE — 어떤 지표를 선택할까?

| 지표 | 핵심 특징 |
| --- | --- |
| **MAE** | 오차의 절댓값 평균. 직관적이고 이상치에 강하다 |
| **MSE** | 오차의 제곱 평균. 큰 오차에 훨씬 민감하다 |

- **MSE 선택** → 큰 오차가 치명적인 도메인 (의료, 금융)
- **MAE 선택** → 이상치가 많거나 오차를 실제 단위로 표현하고 싶을 때 (부동산 가격 예측, 배달 시간 예측)

> **나의 선택** : MAE로 전반적인 오차 수준을 파악한 뒤, 큰 오차가 중요한 도메인이라면 MSE(또는 RMSE)를 함께 보는 방식을 선호한다.

### Q3. Accuracy만으로 평가하면 안 되는 상황은?

- **클래스 불균형 (Class Imbalance)** 상황
    - 예) 암 진단에서 양성 5%, 음성 95%일 때 — 전부 음성으로 예측해도 Accuracy 95%가 나오지만 쓸모없는 모델

- **대신 사용하는 지표**

| 지표 | 설명 | 언제 유용한가 |
| --- | --- | --- |
| **Precision** | 양성 예측 중 실제 양성 비율 | 스팸 필터 등 FP를 줄이고 싶을 때 |
| **Recall** | 실제 양성 중 맞춘 비율 | 암 진단 등 FN을 줄이고 싶을 때 |
| **F1 Score** | Precision과 Recall의 조화평균 | 둘 다 균형 있게 볼 때 |

- **핵심**: Precision ↑ 이면 Recall ↓ , 서로 트레이드오프 관계
    - 암 진단 → **Recall 우선** (놓치면 안 됨)
    - 스팸 필터 → **Precision 우선** (정상 메일이 차단되면 안 됨)

→ 도메인에 따라 어떤 오류가 더 치명적인지 먼저 판단하고 지표를 선택해야 한다.
